CPU 파이프라이닝
- CPU는 한 번에 한 명령어씩 실행하는 것이 아님.
- 한 작업이 끝나기 전에, 다음 작업을 시작하는 방식으로
  동시에 여러 개의 작업을 실행하는 것을
  파이프라이닝이라고 함.
- CPU에서 명령어를 실행할 때 여러 단계를 거침
	- 명령어를 읽기 Fetch
	- 읽은 명령어가 무엇인지 해석 Decode
	- 해석된 명령어를 실행 Execute
	- 명령어 결과를 쓰기 Write
- 컴파일러는 최대한 CPU의 파이프라인을
  효율적으로 활용할 수 있도록 명령어를 재배치함.
- 각 코어가 각각 자신들의 L1, L2 캐시들을 가지고 있음.
  그래서 동기화 과정을 거치지 않는다면 다른 쓰레드에서 알 수 없음.


원자성
- 원자적 연산
	- CPU가 명령어 1개로 처리하는 명령
	- 중간에 다른 쓰레드가 끼어들 여지가 전혀 없는 연산을 말함.
	- 즉 이 연산을 반 정도 했다는 있을 수 없고,
	  이 연산을 했다 / 안 했다로만 구분할 수 있음을 의미함.
- 이처럼 원자처럼 쪼갤 수 없다고 해서 원자적 Atomic 이라고 함.
